# История изменений (Changelog) - Солнечный компас
## [1.0.6] - Умный расчет будущего времени & Build 1.0.6
**Исправлено:**
* Исправлена логика поиска времени инсоляции: алгоритм теперь проверяет условие `passageDate > now`. Если солнце в выбранном азимуте сегодня уже прошло, приложение автоматически переключается на поиск этого же положения в будущие дни (например, завтра в то же время).
## [1.0.5] - Чистая 2D проекция (Вид сверху)
* Переход на единую 2D систему координат (вид сверху, Земля в центре). Весь диск компаса (север, деления и маркеры солнца) вращается как единый плоский прибор.
* **Текущее Солнце**: отображается белой точкой на внутреннем орбитальном кольце ($r = 100$), показывая реальное направление на солнце в текущий момент суток.
* **Целевое Солнце**: фиксируется на внешнем кольце ($r = 140$) при нажатии кнопки спуска.
* Добавлена верхняя визирная риска-индикатор для точного наведения в стиле видоискателя.
## [1.0.4] Добавляем орбитальное кольцо траектории солнца, убираем лишнюю надпись «Наведитесь» и сохраняем север на месте.
В этой сборке (Build 1.0.4) на компас добавлено второе внутреннее орбитальное кольцо с плавающей меткой реального положения солнца на текущий момент дня. Настоящий север (N) остаётся на своем месте, а кольцо наглядно показывает, где солнце находится прямо сейчас относительно вашего направления.
## [1.0.3] - Поиск без разрывов & Build Tracker
**Исправлено:**
* Критический баг "00:00", возникавший из-за бинарного поиска на стыке углов $359^\circ$ и $0^\circ$.
* Алгоритм расчёта инсоляции полностью переписан на комбинированный метод: сканирование суток с шагом 30 минут + поминутный Brute-Force (`±45` минут) при нахождении нужного сектора.
* `findSolarPassage` теперь стартует ровно в `00:00` текущего дня, чтобы не пропускать утренние часы.

**Добавлено:**
* Трекер версии приложения (`Build 1.0.3`) внизу экрана в стиле Ghost / Apple (минималистичная типографика, $10\text{px}$, opacity $15\%$).

## [1.0.2] - Shutter Button & Camera UX
**Добавлено:**
* Интерфейс переведён в парадигму приложения "Камера". Кнопка "Рассчитать" заменена на визуальную кнопку спуска затвора (Shutter Button).
* Shutter Freeze: нажатие на компас или кнопку спуска блокирует азимут намертво. Сдвиг телефона больше не сбрасывает результат. Сброс происходит только по повторному тапу.
* Визуальный отклик фиксации: кнопка спуска преобразуется в янтарный индикатор при успешном наведении.

**Исправлено:**
* Проблема сброса результатов от микро-движений рук ("шум" гироскопа iPhone).

## [1.0.1] - Apple Minimalist UI & PWA Fixes
**Добавлено:**
* Полностью новый дизайн (Glassmorphism, Dark Graphite, типографика UltraLight $72\text{px}$). 
* Стартовый экран-оверлей (`startOverlay`) для корректного запроса прав гироскопа и геолокации (обход ограничений iOS Safari / режима Home Screen).
* Бесшовное вращение компаса (Unwrapped Heading), предотвращающее скачок кольца при переходе через север.

**Исправлено:**
* Огромная нагрузка на CPU (Throttling и перенос вычислений в `requestAnimationFrame`).
* Интегрирована астрономическая модель NOAA для точного звездного времени (GMST).
